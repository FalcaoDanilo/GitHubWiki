Sobre o GitHub

O GitHub é uma plataforma de hospedagem de código fonte e colaboração que permite que desenvolvedores trabalhem juntos em projetos. Ele oferece recursos para versionamento de código, controle de alterações, gerenciamento de problemas e muito mais.

Com o GitHub, é possível colaborar com outros desenvolvedores, compartilhar e revisar código, além de acompanhar o histórico de alterações de um projeto.

Utilização do GitHub

  1. Passo 1: Configurar o Git

    Antes de começar a usar o GitHub, você precisa configurar o Git em seu computador. Abra o terminal e execute os seguintes comandos:

    git config --global user.name "Seu Nome"
    git config --global user.email "seu-email@example.com"

  2. Passo 2: Criar um Repositório

    Para criar um novo repositório no GitHub, siga estes passos:

    1. Acesse o GitHub e faça login na sua conta.
    2. Clique no botão "New" ou "Novo" para criar um novo repositório.
    3. Preencha o nome do repositório, uma descrição opcional e escolha as configurações desejadas.
    4. Clique em "Create Repository" ou "Criar Repositório" para criar o repositório.

  3. Passo 3: Clonar um Repositório

    Para clonar um repositório existente do GitHub para o seu computador, siga estes passos:

    git clone https://github.com/nome-do-usuario/nome-do-repositorio.git

Comandos do Git

  • git init: Inicializa um repositório Git
  • git add: Adiciona arquivos ao índice de mudanças
  • git commit: Cria um novo commit com as mudanças adicionadas
  • git push: Envia as alterações para o repositório remoto
  • git pull: Atualiza o repositório local com as alterações do repositório remoto
  • git branch: Lista, cria ou deleta branches
  • git checkout: Muda para uma branch ou restaura arquivos
  • git merge: Combina mudanças de uma branch com outra
  • git stash: Armazena as mudanças em um stash temporário
  • git cherry-pick commit: Aplica as alterações de um commit específico

Outros Comandos Importantes

  • git status: Mostra o status atual do repositório
  • git log: Exibe o histórico de commits
  • git branch -D branch: Deleta uma branch local
  • git remote add origin remote-url: Adiciona um repositório remoto
  • git remote -v: Lista os repositórios remotos configurados

Dicas de commits

     git commit -m "<'tipo>[escopo opcional]:<'descrição>"
  • FIX: Resolve um bug
  • FEAT: Inicia a implementação de uma funcionalidade
  • CHORE: Trabalho em progresso de uma funcionalidade
  • REFACTOR: Ajuste de uma funcionalidade sem mudar a lógica - refatoração
  • TEST: Implenta testes automatizados
  • STYLE: Mudança de formatação do código - LINT
  • PERF: Ajustes de performance
  • DOCS: Insere documentação
  • CI: Ajustes nas configurações do CI
  • BUILD: Ajuste nas configurações de build
